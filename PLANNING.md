# 服薬記録アプリ プランニング

## 1. アプリの概要

**目的**: 毎日薬を飲んだかどうかを記録し、飲み忘れを防ぎ、継続日数（ストリーク）を可視化する。

**想定ユーザー**: 毎日決まった薬を服用する人（慢性疾患・サプリ・習慣化したい人など）

---

- **起動**: アプリを開くと最初からこのカレンダー画面が表示される（スプラッシュやオンボーディングなし、または最小限）。

---

## 3. 機能一覧

| 機能 | 説明 |
|------|------|
| カレンダー表示 | 月単位でカレンダーを表示。日付セルで「飲んだ」「飲んでいない」を視覚的に区別。 |
| タップで記録 | 日付をタップするとその日を「飲んだ」に記録。再度タップで「飲んでいない」に戻すかは仕様で決定（トグル or 飲んだのみ）。 |
| 連続服用日数（ストリーク） | 今日を含めて何日連続で飲んでいるかを表示。**仮データ**: 115日からスタート。 |
| 上段メーター/数字 | ストリークを数字やプログレスバー・バッジで表示。 |
| 下段メーター/数字 | 同じストリーク、または「今月の服用日数」などを表示。 |
| アラート1 | ボタン押下で「飲み忘れてますよ」系のアラート（ダイアログ）を表示。サンプル用。 |
| アラート2 | ボタン押下で「薬が明日なくなります。補充してください。」のアラートを表示。サンプル用。 |

---

## 4. データモデル（案）

- **日付 → 服用有無**: `[日付: 飲んだか]` の辞書または日付の配列。
- **ストリーク計算**: 「今日」から遡って連続で「飲んだ」が続く日数を計算。  
  - 仮で115日からスタートする場合は、過去115日分を「飲んだ」として持つか、ストリーク値だけを定数で115にしておく。

```swift
// イメージ
struct MedicationRecord {
    var takenDates: Set<Date>  // 飲んだ日付の集合
    var currentStreak: Int     // 現在の連続日数（仮で115から開始）
}
```

---

## 5. 画面フロー

1. アプリ起動 → **メイン（カレンダー）画面**
2. カレンダーで日付タップ → その日を「飲んだ」に記録（またはトグル）
3. ストリークは記録に応じて再計算し、上段・下段に更新表示
4. 「アラート1」タップ → アラートダイアログ「飲み忘れてますよ」表示
5. 「アラート2」タップ → アラートダイアログ「薬が明日なくなります。補充してください。」表示

---

## 6. デザイン方針（スタイリッシュ）

- **カラー**: ダークテーマ or ライトテーマのどちらか統一。アクセントは1色（例: 青・緑・ティール）に絞る。
- **フォント**: システムフォント（SF Pro）をベースに、数字はやや大きめ・太めでストリークを強調。
- **カレンダー**:
  - 飲んだ日: チェックマーク、丸、色付き背景などで明確に。
  - 飲んでいない日: グレーまたは薄い色、もしくは「－」「未」など短いラベル。
- **余白**: 十分なパディングでごちゃつかないように。
- **アラートボタン**: プライマリ／セカンダリで色分けし、押したら標準のアラート（UIAlertController）でメッセージ表示。

- **プラットフォーム**: iOS（iPhone）
- **言語**: Swift
- **UI**: SwiftUI 推奨（カレンダーは UIKit の `UICalendarView` を SwiftUI でラップするか、SwiftUI でカスタムグリッドで実装）
- **データ永続化**: UserDefaults または SwiftData / Core Data（将来拡張を考えると SwiftData がおすすめ）
- **ストリーク計算**: 今日から遡って `takenDates` を連続でチェックする関数を用意

## 8. 仮データ・初期状態

- **ストリーク**: 115日で固定スタート（または過去115日分を「飲んだ」として持つ）。
- **カレンダー**: 当月を表示し、過去115日分に「飲んだ」マークが入っている状態で表示すると分かりやすい。
