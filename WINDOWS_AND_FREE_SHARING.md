# Windows で開発して、無料で友達の iPhone で触れる方法

Mac がなくても、**Windows だけで開発**し、**無料**で**友達の iPhone で実際に操作感・使い勝手を試せる**方法をまとめます。

---

## おすすめ：2つの方法

| 方法 | 開発環境 | iPhone で試す方法 | 友達との共有 | 操作感 |
|------|----------|-------------------|--------------|--------|
| **PWA** | Windows（ブラウザ＋エディタ） | Safari で開いて「ホーム画面に追加」するとアプリのようにフルスクリーンで使える | URL を送るだけ | ウェブだが、アプリっぽく使える |
| **Expo（React Native）** | Windows（Node.js + エディタ） | **Expo Go** アプリで QR コード読み取り → その場でアプリが起動 | 同じ URL/QR を送る | ネイティブに近い |

どちらも **Mac 不要・追加費用なし** で、**友達は自分の iPhone で実際に触れます**。

---

## 方法1：PWA（Progressive Web App）で作る

### どんなものか

- **Web サイト**として作る（HTML/CSS/JavaScript、または React など）。
- スマホのブラウザ（Safari）で開き、**「ホーム画面に追加」** すると、**アプリのようにフルスクリーン**で起動し、オフラインでもある程度動かせるようにできる。

### 必要なもの（すべて無料）

| 項目 | 内容 |
|------|------|
| **PC** | Windows（今お使いの環境） |
| **エディタ** | VS Code など（無料） |
| **開発** | ブラウザ（Chrome / Edge）で表示確認 |
| **公開** | **Vercel** や **Netlify** や **GitHub Pages**（無料） |

### 友達が iPhone で試す手順

1. あなたが PWA を **Vercel / Netlify / GitHub Pages** で公開（無料）。
2. **URL** を友達に送る（LINE / Slack など）。
3. 友達が **iPhone の Safari** でその URL を開く。
4. Safari の **共有ボタン** → **「ホーム画面に追加」** をタップ。
5. ホーム画面に追加されたアイコンをタップすると、**ブラウザの UI がなくなり、アプリのように** 使える。

### メリット・注意点

- **メリット**: Mac 不要、完全無料、URL 共有だけで誰でも iPhone で試せる。  
- **注意点**: ネイティブアプリほどではないが、タップ・スワイプ・カレンダーなどは十分再現でき、**操作感や使い勝手は実機で体感できる**。

---

## 方法2：Expo（React Native）＋ Expo Go で作る

### どんなものか

- **React Native** でアプリを作り、**Expo** というツールでまとめる。
- **Expo Go** という無料アプリ（App Store からインストール）を iPhone に入れ、**QR コード** や **URL** であなたのプロジェクトに接続すると、**その場でアプリが起動**する（ネイティブに近い動き）。

### 必要なもの（すべて無料）

| 項目 | 内容 |
|------|------|
| **PC** | Windows |
| **Node.js** | nodejs.org からインストール（無料） |
| **エディタ** | VS Code など（無料） |
| **Expo アカウント** | 無料で作成（expo.dev） |
| **iPhone 側** | **Expo Go** アプリ（App Store で無料） |

### 友達が iPhone で試す手順

1. あなたが Windows で Expo プロジェクトを開発。
2. **Expo のクラウド** にビルドを公開（`expo publish` や EAS Update の無料枠など）。
3. 友達の iPhone に **Expo Go** をインストール（無料）。
4. あなたが **共有用 URL** または **QR コード** を友達に送る。
5. 友達が Expo Go でその URL を開くか、QR を読み取る → **あなたのアプリがそのまま起動**し、実機で操作できる。

### メリット・注意点

- **メリット**: ネイティブに近い操作感。Windows だけで開発でき、友達は Expo Go を入れるだけで同じものを iPhone で試せる。無料枠で共有可能。  
- **注意点**: 友達は「Expo Go」というアプリを経由して開く形になる（ストアに配布するような「1本のアプリ」ではない）。

## 服薬記録アプリの場合、どちらが向いているか

| 観点 | PWA | Expo |
|------|-----|------|
| カレンダー・タップで記録 | 十分実現可能 | 十分実現可能 |
| ストリーク表示・アラート | 可能 | 可能 |
| 操作感を「実機で」確認 | アプリっぽく使える | よりネイティブに近い |
| 設定の手軽さ | ブラウザ＋エディタだけでOK | Node.js + Expo の導入が必要 |
| 友達に渡す手軽さ | URL 1つでOK | Expo Go インストール＋URL/QR |

- **まず手軽にやりたい・URL だけで共有したい** → **PWA**  
- **よりアプリに近い操作感を重視したい** → **Expo + Expo Go**

---

## まとめ（あなたの条件に当てはめると）

- **Windows で作る** → PWA も Expo も **どちらも Windows だけで開発可能**。  
- **友達との共有は無料** → PWA は **URL 共有のみ**、Expo は **Expo Go（無料）＋ 無料の公開** で共有。  
- **実際に iPhone で操作感・使い勝手が分かる** →  
  - PWA：**Safari で「ホーム画面に追加」** すれば、フルスクリーンでアプリのように触れる。  
  - Expo：**Expo Go** でそのまま実機で触れる。

このどちらかで進めれば、Mac がなくても、無料で、友達の iPhone で「実際の操作感」を体験してもらえます。  
次のステップとして「PWA で服薬記録のプロトタイプを作る」か「Expo で作る」かを決めると、具体的な開発手順まで落とし込めます。
